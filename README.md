# 이거챙겨 (EgoChenggyeo)

이거챙겨는 날씨 정보를 “보여주는” 앱이 아니라,
사용자의 출발 시간 맥락에서 전날 대비 기상 변화를 해석해
“오늘 챙겨야 할 행동”을 추천하는 행동 중심 개인화 기상 알림 앱입니다.

## 핵심 개념
- 정보 제공이 아니라 행동 추천
- 사용자 피드백 기반 점진적 보정
- 로그인 없이 디바이스 기반 사용자 식별(device_id)
- Rule 기반 추천 엔진(기본 판단은 규칙으로)

## 제품 원칙
- MVP는 단일 서버 + RDB 기반의 단순 구조를 우선
- 추천의 판단/점수 계산은 규칙 기반으로 유지
- 개인화는 사용자 피드백(수락/거부)으로 천천히 보정
- AI는 보조 역할만(문구/이유 생성), 판단 로직에는 개입 금지

## 아키텍처 개요
- api: 외부 요청 진입점(Controller + DTO)
- domain: 핵심 비즈니스 로직
- infra: 외부 시스템 연동(날씨/푸시 등)
- support: 공통 설정/예외/스케줄링 등

구체적인 레이어 제약과 작업 규칙은 `AGENTS.md`를 따릅니다.

## MVP 범위 밖(하지 말 것)
- 로그인/OAuth/JWT
- 크롤링/뉴스
- Kafka/Redis/분산 큐 등 복잡한 분산 아키텍처
- domain에서 외부 API 직접 호출
